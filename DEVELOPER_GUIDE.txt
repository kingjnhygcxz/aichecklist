### 1. Voice Recognition System

- Location: server/openai.ts

- Features: Task parsing, intelligent suggestions, natural language processing

### 3. Timer & Alarm System
- Location: client/src/components/timer/, client/src/lib/audio.ts

- Location: server/voice-biometric.ts, server/enhanced-voice-biometric.ts

- Features: Pitch analysis, frequency matching, multi-factor authentication

### 5. Real-time Notifications
- Location: server/notification-service.ts, client/src/hooks/useNotifications.ts
- Technology: TanStack Query + polling
- Features: Calendar reminders, achievement notifications, system alerts

### 6. Project Visualization
- Location: client/src/pages/Charts.tsx

- Features: PERT charts, Gantt charts, performance analytics

## PERFORMANCE FEATURES

### Optimization Techniques
- Code Splitting: Route-based lazy loading
- Bundle Optimization: Vite + esbuild for fast builds
- Database: Connection pooling with PgBouncer
- Caching: TanStack Query for server state
- Memory: Smart API polling (only when active)

### Security Features
- Authentication: Voice biometric + session management
- API Protection: Rate limiting, CORS, Helmet.js
- Data Validation: Zod schemas for all inputs
- SQL Injection: Prevented via Drizzle ORM
- XSS Protection: React's built-in sanitization

## DEVELOPER NAVIGATION GUIDE

### Finding Specific Features

To modify voice recognition:
→ client/src/lib/audio.ts (Audio processing)
→ client/src/hooks/useVoice.ts (Voice hooks)
→ client/src/components/voice/ (UI components)

To modify timer functionality:
→ client/src/components/timer/TimerWidget.tsx (Main timer)
→ client/src/hooks/useTimer.ts (Timer logic)
→ client/src/components/timer/AlarmSettings.tsx (Sound selection)

To modify task management:
→ client/src/components/task/ (Task components)
→ server/routes/tasks.ts (Task API)
→ shared/schema.ts (Task database schema)

To modify authentication:
→ server/voice-biometric.ts (Voice auth logic)
→ client/src/hooks/useAuth.ts (Auth state)
→ server/routes/auth.ts (Auth API)

To modify AI processing:
→ server/openai.ts (GPT-4o integration)
→ client/src/components/ai/ (AI UI components)

To modify database:
→ shared/schema.ts (Database schemas)
→ server/db.ts (Connection configuration)
→ server/storage.ts (Data access layer)

### Common Debugging Locations

Database Issues:
→ Check server/db.ts for connection problems
→ Verify shared/schema.ts for schema mismatches
→ Review server/storage.ts for query errors

Authentication Problems:
→ Check server/voice-biometric.ts for voice auth issues
→ Verify client/src/hooks/useAuth.ts for state problems
→ Review browser console for audio permissions

Timer/Audio Issues:
→ Check client/src/lib/audio.ts for sound mapping
→ Verify browser audio permissions
→ Review client/src/hooks/useTimer.ts for timer logic
